## 端口冲突误报修复

**问题**：进程列表为空，但端口冲突检测报 "端口 57321 被占用"。
**根因**：Manager 自身内嵌 helper server，监听在 57321 端口。`lsof` 检测到 Manager 自己的 PID 占用了端口，但 `helper_running` 为 false（Manager 进程被 `self_pid` 跳过，不计入 helper 角色），导致误判为冲突。
**修复**：端口冲突检测中过滤掉 Manager 自身 PID，只检查"外部进程"占用端口的情况。

**追加修复**：`helper_running` 也应检测 Manager 自身 PID 是否在端口上。如果 `lsof` 发现 Manager 自己的 PID 占用了 helper 端口，说明内嵌 helper 正在运行，应设置 `helper_running = true`。否则前端始终显示"协议代理状态：未运行"。

## 配置管理保存后显示空白修复

**问题**：新增 model_providers 配置后，TOML 文件写入正确，但 UI 显示"无 model_providers 配置"。
**根因**：`save_codex_model_providers` 等 4 个函数使用 `format!("{parent}.{child}")` + `table_mut_or_insert` 创建子表。`table_mut_or_insert` 将 `"model_providers.custom"` 作为字面键名，TOML 输出为 `["model_providers.custom"]`（带引号），重载后被解析为顶级键而非嵌套表，导致 `doc.get("model_providers")` 找不到。
**修复**：改为先创建/获取父表，再在父表内创建子表：
```rust
let parent = table_mut_or_insert(&mut doc, "model_providers")?;
parent[provider_id] = toml_edit::table();
let table = parent.get_mut(provider_id).and_then(Item::as_table_mut)?;
```
影响范围：`save_codex_model_providers`、`save_codex_mcp_servers`、`save_codex_plugins`、`save_codex_projects` 四个函数全部修复。
